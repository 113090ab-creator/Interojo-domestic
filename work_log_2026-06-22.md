# 작업 로그 - 2026-06-22

## 대상
- 국내 생산·포장 현황 대시보드
- 주요 파일: `dashboard.py`, `.streamlit/config.toml`
- 주요 데이터: `생산요청등록(국내).xlsx`, `포장설비투입현황.xlsx`, `수요정보(전공정).xlsx`, `용마WMS재고현황.xlsx`

## 진행 내용

### 1. Streamlit 주소 및 검토 방식 정리
- 로컬 Streamlit 주소(`localhost`)와 사용자가 부여한 앱 주소의 차이를 설명했다.
- ChatGPT가 직접 확인하려면 외부에서 접근 가능한 배포 URL이 필요하고, 로컬 앱은 소스코드와 데이터 기준으로 검토해야 한다고 정리했다.
- 코드 검토 요청 시 필요한 자료를 안내했다.

### 2. 포장 진도율 기준 재정의
- 기존 `생산요청물량 대비 포장실적` 중심의 진도율을 재검토했다.
- 월 단위 메인 진도는 `용마입고수량 / 요청물량` 기준으로 보는 방향으로 정리했다.
- 세부 화면에서는 포장실적과 용마입고를 LOT 단위로 비교하도록 기준을 잡았다.
- 포장은 되었지만 용마입고가 확인되지 않은 수량은 `입고대기`로 분류하는 방향을 반영했다.
- `이전재고 입고`, `미매칭 입고` 등 별도 수량 표시는 제외했다.

### 3. 용마이동현황 및 샘플신청가능수량 반영
- `포장설비투입현황.xlsx`에 추가된 `용마이동현황` 시트를 활용하도록 기준을 정리했다.
- `샘플신청가능수량` 컬럼을 반영했다.
- `포장대기(PCS)` 산식을 아래 기준으로 정리했다.
  - `포장대기(PCS) = 생산요청등록물량 - 수요정보의 누수규격검사 기준 생산수량 + 샘플신청가능수량`
- 품목코드 기준으로 비교해 판매코드 상세에 반영하도록 했다.

### 4. 메인/상세 화면 지표 및 표 구성 조정
- 메인 화면의 포장 진도 표현을 용마입고 기준으로 변경하는 방향을 반영했다.
- 국내 요청 물량 기준의 데이터만 보이도록 범위를 정리했다.
- 포장 LOT 상세는 LOT별 입고대기/입고완료 확인용으로 두고, 생산요청물량·용마입고수량·용마입고대기수량·포장대기수량은 판매코드 상세에서 보는 방향으로 정리했다.
- 긴급 포장 리스트는 용마 보유 재고를 긴급도 정렬 기준으로만 사용하고, 표에는 재고 관련 컬럼이 보이지 않도록 조정했다.
- 생산코드 상세의 `병목 상태` 컬럼을 삭제했다.
- 메인 페이지 제품 진도 현황의 `상태` 컬럼을 삭제했다.

### 5. PACK/PCS 기준 정리
- PACK과 PCS가 한 표에 섞일 때 혼란이 생기는 부분을 검토했다.
- 요청물량은 PACK 기준 조회 시 PACK으로, PCS 기준 조회 시 PCS 환산값으로 볼 수 있도록 방향을 정리했다.
- 포장대기와 생산부족은 원천 데이터 성격상 PCS 기준 의미가 강하므로, 필요 시 PACK 환산은 보조 지표로 다루는 방향으로 정리했다.

### 6. 제품 분류 및 카드 배치 변경
- 메인 제품 카드 영역을 `1DAY`, `FRP`, `기타` 섹션으로 나누어 배치했다.
- `O2O2 D 컬러`, `O2O2 D Micelia`, `O2O2 D Toric`, `Iris 컬러`, `Iris Toric`을 1DAY로 분류했다.
- `PIA 1Day`, `PIA Monthly`는 기타 라인으로 이동했다.
- 기존 카드형 UI는 유지하되 섹션 배치만 업무 기준에 맞게 조정했다.

### 7. POWER 표시 형식 통일
- 대시보드에 표시되는 모든 POWER 값을 `-00.00` 형태로 통일했다.
- 예: `-2.00`은 `-02.00`, `-0.50`은 `-00.50` 형태로 표시되도록 정리했다.

### 8. 미입고 TOP10 기준 정리
- 메인 페이지 `미입고 TOP10` 기준을 설명했다.
- 기준은 제품명 단위로 `미입고수량 = 요청 PACK - 용마입고 PACK`을 계산한 뒤 0 미만은 0으로 보정하고, 미입고수량이 큰 순서로 상위 10개를 보여주는 방식이다.
- 진행률 막대는 용마입고율 기준으로 해석하도록 정리했다.

### 9. PPT 보고서 다운로드 개선
- PPT 다운로드 결과물이 바로 보고서로 사용할 수 있도록 레이아웃을 정리했다.
- PPT 폰트를 `Noto Sans KR`로 변경했다.
- 제목, 기준 시각, KPI 카드, 우선순위 표, 기준 설명 영역의 크기와 배치를 보고서 형태에 맞게 조정했다.
- 표 기준은 `미입고 PACK`, `생산부족 PCS`, `요청 PACK` 중심으로 정렬되도록 정리했다.

### 10. 디자인 시스템 적용
- `interojo_design_system_v2.md` 내용을 기준으로 대시보드 디자인을 적용했다.
- 주요 색상, 배경, 카드, 탭, 버튼, 입력창, 데이터프레임, 진행률 막대, 상태 배지 스타일을 전반적으로 정리했다.
- Plotly 차트 색상과 POWER 히트맵 색상도 디자인 시스템 기준에 맞췄다.
- 기존 `st.metric()` 사용 영역은 커스텀 KPI 카드 형태로 정리했다.

### 11. 와이드 레이아웃 및 Streamlit 테마 설정
- `interojo_design_system_v2 (1).md`의 섹션 8-1 기준을 반영했다.
