## 대시보드 방향성 정리
- 이 대시보드는 국내 요청 물량이 `생산 -> 포장 -> 용마 입고`까지 정상적으로 진행되는지 확인하는 운영 모니터링 화면으로 정의했다.
- 사용자가 화면을 열었을 때 가장 먼저 봐야 하는 것은 “오늘 무엇을 대응해야 하는가”로 정리했다.
- 메인 화면은 전체 현황을 빠르게 판단하고, 상세 탭과 팝업은 원인 확인 및 세부 대응을 위한 구조로 가져간다.
- PACK과 PCS 단위가 혼동되지 않도록 KPI와 컬럼명에 단위를 명확히 표시하는 방향으로 정리했다.

- 요청 대비 용마 입고 현황 카드에 요청 PACK, 용마입고 PACK, 미입고 PACK, 용마입고율을 함께 표시하도록 정리했다.

### 2. KPI 명칭 및 단위 통일
- 아래 명칭을 기준으로 통일했다.
  - 생산진도율
  - 포장진도율

  - 생산부족 PCS
  - 포장부족 PACK
  - 입고대기 PACK

  - 긴급 대응 품목 수
- PACK/PCS 혼동을 줄이기 위해 수량 KPI에는 단위를 포함하도록 정리했다.

### 3. 본품/샘플 KPI 분리
- 전체 KPI 외에 본품 KPI와 샘플 KPI를 분리해서 볼 수 있도록 구성했다.
- 본품과 샘플은 진행률과 부족 리스크가 다르게 해석될 수 있으므로, 전체 현황과 별도로 확인하는 방향으로 정리했다.

### 4. 본품 분류별 진도현황 카드 단순화
- 카드에서 상태와 미입고 수량을 제거했다.
- 카드 표시 항목을 제품분류명, 요청 PACK, 생산진도율, 용마입고율, 생산부족 PCS 중심으로 단순화했다.
- 미입고 수량은 상단 KPI와 미입고 TOP10에서 확인하도록 역할을 분리했다.

### 5. 미입고 TOP10 개선
- 미입고 TOP10은 제품별 미입고 PACK과 생산/포장/용마 입고 진도율을 함께 확인하는 구조로 변경했다.
- 최초에는 추정 원인 컬럼을 추가했으나, 화면 복잡도가 높아져 제거했다.
- 현재 목적은 미입고 규모가 큰 제품을 먼저 확인하고, 진도율 조합으로 원인을 판단하는 것이다.

### 6. 생산/입고 GAP TOP10 개선
- 명칭을 “생산완료 후 미입고 TOP10”으로 변경했다.
- GAP 계산식은 `생산진도율 - 용마입고율`로 정리했다.
- 생산은 진행되었지만 용마 입고가 지연되는 품목을 찾는 목적의 화면으로 정의했다.

### 7. 요청 긴급 요약 개선
- 요청 긴급 요약 컬럼을 다음 순서로 정리했다.
  - 품목코드
  - 제품명
  - 현재 재고수량
  - 부족수량
  - 포장가능재고 PCS
  - 대응가능 여부
- 대응가능 여부 기준을 다음처럼 정의했다.
  - 포장가능재고 >= 부족수량: 충당 가능
  - 포장가능재고 < 부족수량: 일부 가능
  - 포장가능재고 없음: 대응 필요
- 재고수량이 음수인 경우 강조 표시하도록 방향을 정리했다.

## 생산코드 상세 화면 개선

### 1. 메인 테이블 집계 기준 변경
- 기존 생산코드 + POWER 단위 전체 행 표시 방식은 복잡하므로, 생산코드 앞 5자리 기준으로 집계하도록 변경했다.
- 예시:
  - `P3405A-02.00SPC` -> `P3405`
  - `P0176A-02.75MBN3` -> `P0176`
- 메인 테이블은 생산코드 5자리 단위로 제품군 위험도를 빠르게 확인하는 목적이다.

### 2. 메인 테이블 컬럼 정리
- 메인 테이블 컬럼은 선택, 생산코드, 대표 제품명, PACK 단위별 수량, 요청합계 PACK, 포장부족 PACK, 생산부족수량 PCS, 포장진도율, 생산진도율, 최소 납기일 중심으로 정리했다.
- 상태 컬럼은 메인 테이블과 팝업 테이블 모두에서 제거했다.
- PACK 구성은 처음에는 한 컬럼 요약으로 구성했으나, 비교가 어렵다는 판단에 따라 1P, 2P, 5P, 6P, 10P, 30P, 40P, 80P, 90P 단위별 컬럼으로 다시 표시하는 방향으로 변경했다.

### 3. 정렬 기준
- 메인 테이블과 팝업 테이블의 기본 정렬은 동일하게 유지한다.
  - 1순위: 최소 납기일 오름차순
  - 2순위: 포장부족 PACK 내림차순
  - 3순위: 생산부족수량 PCS 내림차순

### 4. 생산코드 필터 기준
- 생산코드 상세 화면의 메인 테이블은 P로 시작하는 생산코드만 표시하는 기준으로 정리했다.
- S 계열처럼 생산코드 상세 목적과 맞지 않는 값은 제외 대상이다.

## 팝업 표준

생산코드 상세에서 적용한 팝업 방식을 이후 상세 조회 화면의 표준으로 삼는다.

### 1. 팝업 사용 목적
- 메인 화면은 집계/요약 중심으로 유지한다.
- 행 클릭 또는 선택 시 팝업에서 상세 내역을 보여준다.
- 상세 정보가 많아 메인 테이블을 복잡하게 만드는 경우 팝업으로 분리한다.

### 2. 팝업 제목 규칙
- 제목은 “무엇의 상세인지”와 “대표 제품명”이 바로 보이도록 구성한다.
- 예시:
  - `생산코드 P3405 상세 - Clalen O2O2 D_Sepia Choco EX`

### 3. 팝업 테이블 규칙
- 팝업은 선택한 집계 행의 하위 상세 데이터를 보여준다.
- 메인 테이블에서 숨긴 세부 컬럼을 팝업에서 노출한다.
- 숫자는 우측 정렬하고 천 단위 콤마를 적용한다.
- 퍼센트는 소수점 1자리로 표시한다.
- 상태 컬럼처럼 판단에 큰 도움이 없거나 중복되는 컬럼은 제외한다.
- 정렬 기준은 메인 테이블과 동일하게 유지한다.

### 4. 팝업 UI 규칙
- 닫기 버튼으로 닫을 수 있어야 한다.
- 과한 색상, 그림자, 큰 여백은 사용하지 않는다.
- 표준 테이블 스타일을 사용하고, 위험 수치만 강조한다.
- 팝업은 상세 확인 용도이며 KPI나 주요 판단 정보는 메인 화면에서 먼저 보이게 한다.
